⚡ Phase 2: Cryptographically Signed Service Requests

Status: 40% complete • Swap & Pools live, Send/Bridge/AI/IoT/Music in progress

Architecture: Centralized Signed Requests
  • Frontend: walletV1BuildSignedRequest() builds canonical payload with timestamp, nonce, signature
  • Signature Format: secp256k1_compact (128 hex chars, ECDSA SHA256)
  • Public Key Format: Compressed (02/03 + 64 hex) or uncompressed (04 + 128 hex)
  • Validation: Client-side format check + backend signature verification
  • Binding Check: Backend verifies public_key_hash against active bindings
  • Direct Wallets: Signer == canonical still supported for direct V1 wallets

Miner Kit Architecture
  • Pledge System: New miners pledge THR → canonical_v1_address as reward destination
  • Worker Format: <canonical_v1_address>.<worker_name> (e.g., THR6...@worker1)

  • Reward Split: Pool fee + reward_to_miner + reward_to_ai + fee_burned
